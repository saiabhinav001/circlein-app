# Community Section Admin Operations Guide

This guide explains the fastest and safest way to run the Community section.

## Daily Workflow

1. Open Community Hub and check pinned announcement status.
2. If there is an outdated pinned post, unpin it.
3. Publish new announcements only when they are clear and actionable.
4. Review active polls and ensure deadline windows are still relevant.
5. Scan latest comments and post clarifications when needed.

## Announcement Checklist

Use this checklist before publishing:

- Title says what changed in one line.
- Body includes date, time, impact, and next step.
- If there is service disruption, include alternatives.
- Keep sensitive user information out of announcements.

Recommended structure:

- What changed
- When it starts/ends
- Who is affected
- What residents should do
- Contact path

## Poll Checklist

Use this checklist before publishing:

- Question asks one clear decision.
- Options do not overlap.
- Deadline is in the future and gives enough response time.
- Poll is relevant to residents and operationally actionable.

## Pin/Unpin Rule

- Keep one critical pinned announcement at a time.
- Unpin resolved posts quickly.
- If two urgent updates exist, merge them into a single pinned summary.

## Incident Communication Template

Title: Planned maintenance update

Body:

### Heads up

We will perform scheduled maintenance on [amenity] on [date] between [time].

- Expected impact: [brief impact]
- Alternative options: [other amenities]
- Contact: [name/channel]

Thanks for your cooperation.

## Poll Template

Question: Should we extend weekend amenity hours?

Options:

- Yes, extend by 1 hour
- Keep current timing

## Governance Notes

- Keep communication factual and neutral.
- Do not use accusatory language in public posts.
- Confirm details with operations team before posting urgent alerts.

## Troubleshooting

- If publish fails, retry once and confirm internet/session.
- If pin update fails, refresh and try again.
- If poll creation fails, verify deadline format and options count.

If issues persist, contact the technical admin.
