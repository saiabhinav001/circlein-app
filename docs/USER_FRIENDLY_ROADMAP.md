# User-Friendly Roadmap (All Age Groups)

## Product Principle
Make every daily task possible in under three taps, using plain language and clear actions.

## Brutal-Honest Decision: Keep or Remove SLA Risk?
- Decision: Keep the SLA engine, but hide the jargon by default.
- Reason: Auto-escalation prevents missed support cases and improves reliability.
- UX rule: Show plain labels in Simple Mode (for most admins). Keep advanced radar only when explicitly enabled.

## Navigation Order (Recommended)

### Resident order

### Admin order

8. Insights (Advanced)

## What to Simplify or Remove from Default View
- Hide technical labels by default: SLA, triage, risk index, escalation reason codes.
- Replace with plain labels: Support follow-up needed, Open tasks, Bookings next 7 days.
- Keep advanced analytics page accessible through explicit Advanced toggle.

## High-Impact Accessibility Features (Phase 2)
1. Large Text Mode (persistent, one-tap).
2. Voice-guided quick actions for admin home.
3. Confirm-before-danger actions with clear undo text.
4. Screen-reader pass on all forms and dialogs.
5. One-screen quick-start checklist for newly trained admins.

## Premium and Differentiating Features (Phase 3)
1. Operator Copilot panel that explains next best action in plain language.
2. "What changed today" digest card with one-click navigation to affected pages.
3. Smart incident lane: auto-group related complaints and maintenance items.
4. Local-language pack switcher with role-based defaults.

## Responsiveness and Visual Placement Rules
- Minimum tap target: 44x44 px for all icon-only buttons.
- Main action always visible above fold on mobile.
- Keep list pages to one primary action per row.
- Avoid icon-only meaning; pair with labels where possible.

## Rollout Plan

### Phase 1 (done in this iteration)
- Simple Mode behavior introduced in admin UX.
- Navigation labels and order simplified.
- Operations health card now has plain-language summary.
- Header and critical icon controls enlarged for touch accessibility.

### Phase 2 (next)

- Add guided task checklist on Admin Home.
- Add inline field help for all admin forms.

### Phase 3 (next)

- Smart incident lane.
